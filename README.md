Задание по анализу данных от компании M.Reason


Шаг 1. Create step_1_anomaly_checking

Выявление системных проблем в первичном источнике данных
1) Отрицательные продажи → ошибки ввода или возвраты без корректного учёта
2) Заглушки (-999) → сбойные ETL-процессы или проблемы с интеграцией
3) Дубликаты → ошибки в системах учёта или ручном вводе

Результат работы:
- anomaly_report.txt
- anomaly_report.xlsx

Данные индикативны, для принятия решения производить первичные правки или нет

Шаг 2. step_2_summing_duplicate_rows
Группировка по ключевым атрибутам:

Месяц продаж → временной период
Год коллекции → сезонность
Магазин → географическое расположение
Коллекция → товарная группа

Результат работы: файл step2.xlsx с суммированными значениями по:

Количество остаток начало
Стоимость остатков в розничных ценах со скидкой начало
Количество остаток средний
Стоимость остатков в розничных ценах со скидкой средний 
Количество продажа
Сумма выручка

Шаг 3. step_3_anomaly_checking
Повторная проверка. Выявление оаставшихся аномалий в результирующем файле файл step2.xlsx. Аналогичная проверке на шаге 1.
Результат проверки выводится в индикативных файлах anomaly_report_2.xlsx и anomaly_report_2.txt
для принятия решения производить первичные правки или нет. В случае отсутствия правок строк с "аномалиями" эти строки  будут игнорироваться при анализе.
